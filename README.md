## Key Lessons on Clean Code

- **Meaningful Naming** — Clear, descriptive names for variables, functions, and classes (e.g., `get_valid_score` instead of `gs`) improve readability and maintainability.
- **Single Responsibility Principle (SRP)** — Every function should do one job only. This makes code easier to test, reuse, and debug.
- **Modular Design** — Breaking programs into small, focused functions reduces complexity and avoids the "wall of code" in `main()`.
- **Error Handling** — Using exception handling (Practical 3 onwards) makes programs robust against bad user input.
- **Constants over Magic Numbers** — Defining values like `MINIMUM_LENGTH = 8` at the top of a file makes changes easy and intent clear.
