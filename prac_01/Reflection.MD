## Reflection – Practical 1

**1. What did you learn about loops and conditionals this week?**  
I learned how `for` and `while` loops work differently — `for` loops are better when you know the count in advance, and `while` loops are better for input validation or sentinel-controlled repetition. I also practised fixing broken boundary conditions in `if/elif` chains to get the logic right.

**2. What was the hardest part?**  
The hardest part was the menu program — keeping the loop running correctly while showing the menu each time and only quitting on `Q`. It was easy to get the structure slightly wrong and either show the menu twice or not at all.

**3. What did you enjoy or do well?**  
I enjoyed the temperature converter because it combined a constant, a menu loop, and two-way conversion cleanly. I also did well using `f-strings` for formatted output consistently across all the programs.

**4. What would you change or do better next time?**  
Next time I will trace through the loop logic on paper before I type it. I wasted time running programs that had slightly wrong loop conditions when a quick trace would have caught it immediately.

**5. How do you feel about using Git and GitHub?**  
Git is still new to me but I can see the value — every commit is like a save point. I want to get better at writing clear commit messages so the history actually makes sense when I look back at it.
